Active Directory : pilier des réseaux Windows en entreprise

2025 SISR – Administration système Enzo Savary

Qu'est-ce que l'Active Directory ?

L'Active Directory (AD) est un service d'annuaire développé par Microsoft, intégré à Windows Server depuis la version 2000. Il repose sur les protocoles LDAP (Lightweight Directory Access Protocol) et Kerberos pour l'authentification et la gestion des objets réseau.

Il permet à une entreprise de gérer de manière centralisée l'ensemble de son parc informatique : comptes utilisateurs, ordinateurs, imprimantes, groupes et stratégies de sécurité depuis une interface unique.

Domaine

Unité administrative regroupant les objets (utilisateurs, machines) partageant une même base de données AD et des politiques de sécurité communes.

Contrôleur de domaine

Serveur hébergeant l'AD DS (Active Directory Domain Services). Il authentifie les utilisateurs et applique les stratégies de groupe sur le domaine.

Unité organisationnelle (UO)

Conteneur permettant d'organiser les objets du domaine en sous-groupes logiques (ex : par département) pour faciliter la gestion et l'application des GPO.

Forêt & Arborescence

La forêt est l'ensemble de domaines partageant un schéma commun. L'arborescence est un groupe de domaines liés par des relations d'approbation.

Les composants clés

AD DS – Active Directory Domain Services

Le rôle principal. Gère l'annuaire des objets (utilisateurs, groupes, ordinateurs) et l'authentification au sein du domaine. C'est le cœur de tout réseau Windows en entreprise.

DNS – Domain Name System

Indissociable de l'AD. Permet la résolution des noms du domaine. Le contrôleur de domaine intègre automatiquement le rôle DNS lors de sa promotion.

GPO – Group Policy Objects

Stratégies de groupe permettant d'appliquer des configurations de sécurité et des restrictions à des groupes d'utilisateurs ou d'ordinateurs (fond d'écran imposé, politique de mots de passe, restrictions logicielles…).

Kerberos – Authentification SSO

Protocole d'authentification utilisé par l'AD. Permet à un utilisateur de se connecter une seule fois (Single Sign-On) pour accéder à toutes les ressources du domaine sans ressaisir ses identifiants.

LDAP – Protocole d'annuaire

Protocole de communication permettant d'interroger et de modifier les données de l'annuaire. Utilisé par de nombreuses applications tierces pour s'authentifier contre l'AD.

Points clés à retenir

Centralisation : un seul point de gestion pour tous les comptes, machines et politiques du réseau d'entreprise.

Sécurité : les GPO permettent d'imposer des règles de sécurité uniformes sur l'ensemble du parc (mots de passe, verrouillage, chiffrement…).

Scalabilité : l'AD supporte des environnements de quelques machines à plusieurs milliers de postes et d'utilisateurs.

Délégation : il est possible de déléguer l'administration de certaines UO à des techniciens sans leur donner les droits complets d'administrateur du domaine.

Intégration : l'AD s'intègre avec Microsoft 365, Azure AD, Exchange, et de nombreux logiciels tiers via LDAP/SAML pour l'authentification unifiée.
