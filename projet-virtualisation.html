Contexte & objectifs

Dans le cadre de la formation BTS SIO SISR, l'utilisation de la virtualisation est fondamentale pour pratiquer l'administration système sans risquer d'endommager un environnement de production. VirtualBox (Oracle) a été utilisé comme hyperviseur de type 2 pour créer et gérer plusieurs machines virtuelles.

L'objectif était de maîtriser la création, la configuration et la gestion de VMs, de comprendre les différents modes réseau de VirtualBox, et d'utiliser les snapshots pour sauvegarder et restaurer l'état d'une machine.

Systèmes déployés : Windows Server 2019 (contrôleur de domaine), Windows 10 (poste client), Ubuntu Server (serveur Linux).

Étapes de réalisation

1

Installation et configuration de VirtualBox

Installation de VirtualBox sur le poste hôte. Installation des Guest Additions pour améliorer les performances et activer le copier-coller entre l'hôte et les VMs.

2

Création de la VM Windows Server 2019

Allocation de 2 Go RAM, 2 CPU, 50 Go disque. Installation depuis l'ISO. Configuration de l'adresse IP statique 192.168.1.1/24. Mode réseau : réseau interne intnet.

3

Création de la VM Windows 10 client

Allocation de 2 Go RAM, 1 CPU, 30 Go disque. Configuration du réseau interne identique pour communiquer avec le serveur. DHCP configuré pour recevoir une IP automatiquement.

4

Configuration des modes réseau

Test des différents modes réseau VirtualBox : NAT (accès internet), réseau interne (communication VM↔VM), pont (accès au réseau physique). Choix du mode réseau interne pour les TP d'infrastructure isolée.

5

Gestion des snapshots

Création de snapshots avant chaque manipulation risquée. Restauration d'un snapshot après une mauvaise manipulation sur le serveur AD. Compréhension de l'arborescence des états de la machine.

Difficultés rencontrées & solutions

Les VMs ne communiquaient pas entre elles

✓ Solution

Les deux VMs n'étaient pas sur le même réseau interne. Uniformisation du nom du réseau interne (intnet) sur les deux machines dans les paramètres réseau VirtualBox.

Performances insuffisantes sur la VM Server

✓ Solution

Activation de la virtualisation matérielle (VT-x/AMD-V) dans le BIOS du poste hôte et augmentation de la mémoire vidéo allouée à la VM. Installation des Guest Additions pour les drivers optimisés.
